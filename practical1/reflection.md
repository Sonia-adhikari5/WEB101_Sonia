# Reflection on Practical 1 – WEB101

This document contains my reflections on the topics covered in this practical. For each section, I have described what I learned, the challenges I faced, and how I overcame them.

---

### 1. Project Setup

### What I Learned

I learned how to create a Next.js project using the terminal and configure it with important features such as ESLint, Tailwind CSS, and the App Router.

I also understood how a project is structured and how different folders like app and components are organized.

### Challenges Faced

I found it confusing to understand the setup options at first, especially when the terminal asked multiple configuration questions.

I was also unsure about how long the installation process should take.

### How I Overcame Them

I carefully followed the instructions and selected the required options.

I waited for the installation to complete and checked for errors in the terminal before proceeding.

---

### 2. Routing and Pages

### What I Learned

I learned how Next.js uses folder-based routing, where each folder represents a page in the application.

I understood how creating a file like page.jsx automatically creates a route such as /profile or /upload.

### Challenges Faced

At first, I was confused about where to create files and how the routing system works.

I also made mistakes with file names and folder placement.

### How I Overcame Them

I followed the correct folder structure and ensured that each page was placed inside the app directory.

Practicing by creating multiple pages helped me understand routing better.

---

### 3. Components and Layout

### What I Learned

I learned how to create reusable components such as MainLayout, VideoCard, and VideoFeed.

I also understood how layout.js is used to apply a common structure across all pages.

### Challenges Faced

I had difficulty understanding how components are connected and how children works in layouts.

I also made mistakes with imports, especially using the correct syntax for Link.

### How I Overcame Them

I reviewed examples and corrected my import statements.

I practiced using components and gradually understood how they fit together.

---

### 4. Styling with Tailwind CSS

### What I Learned

I learned how to use Tailwind CSS utility classes to design the interface.

I was able to style elements such as buttons, layouts, and containers quickly.

### Challenges Faced

At first, the class names were confusing and difficult to remember.

I was unsure how different classes affected the layout.

### How I Overcame Them

I experimented with different Tailwind classes and observed the changes.

With practice, I became more comfortable using them.

---

### 5. Forms and Validation

### What I Learned

I learned how to create forms using react-hook-form.

I understood how validation works, such as required fields and input checks.

### Challenges Faced

I found it difficult to understand how form validation functions work.

I also had trouble handling form submission correctly.

### How I Overcame Them

I followed examples and tested different inputs to see how validation behaves.

This helped me understand how to manage form data properly.

---

### Overall Reflection

This practical helped me understand how modern web applications are built using Next.js and React. I learned how routing, components, and styling all work together.

Although I faced several challenges, especially with project structure and components, I was able to overcome them through practice and debugging.

---

### Future Improvements

I want to improve my understanding of React components and state management.

I also plan to practice building more projects to strengthen my frontend development skills.

In the future, I aim to create more advanced and interactive web applications.
