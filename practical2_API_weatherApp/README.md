# Practical Report – Weather API Dashboard (WEB101)

## Purpose of the Practical

1. This practical focuses on developing a web-based Weather Dashboard using HTML, CSS, and JavaScript.
2. It aims to build understanding of how structure, styling, and interactivity work together in a web application.
3. The practical also introduces the use of JavaScript to handle user input and dynamically update content on a webpage.

---

## Instructions and Solutions

### 1. Project Setup

#### 1.1 Creating the HTML Structure

* **Task:** Create a basic HTML structure for the application.
* **Task:** Add meta tags and a meaningful title.

**Solution:**

The HTML file was structured using standard tags such as `<!DOCTYPE html>`, `<html>`, `<head>`, and `<body>`. Meta tags were added to ensure proper character encoding and responsive design.

---

#### 1.2 Styling with CSS

* **Task:** Design the layout using CSS.
* **Task:** Style containers, buttons, and input fields.

**Solution:**

CSS was implemented inside the `<style>` tag to control layout and appearance. Key styling features include:
- Centered layout using margin and max-width
- Card-style containers with shadows and rounded corners
- Styled buttons with hover effects
- Clean typography and spacing

---

### 2. Layout Design

#### 2.1 Container and Page Layout

* **Task:** Create a structured layout for displaying weather information.

**Solution:**

A container class was used to group related content. The layout was designed to be clean and readable, with proper spacing between sections.

---

#### 2.2 Form Elements

* **Task:** Design input fields and buttons for user interaction.

**Solution:**

Form elements such as inputs, labels, and buttons were styled using CSS. The button includes hover effects to improve user experience.

---

### 3. JavaScript Functionality

#### 3.1 Handling User Input

* **Task:** Allow users to enter a location.

**Solution:**

JavaScript was used to capture user input from form fields and process it when the user submits the form.

---

#### 3.2 Dynamic Content Display

* **Task:** Display weather data dynamically on the page.

**Solution:**

JavaScript was used to update the DOM (Document Object Model). Weather information is displayed using dynamically created elements such as weather cards.

---

#### 3.3 Managing Saved Locations

* **Task:** Store and display multiple locations.

**Solution:**

JavaScript logic was used to manage saved locations and display them using a flexible layout.

---

### 4. UI Components

#### 4.1 Weather Display

* **Task:** Create a section to display weather details.

**Solution:**

Weather information is displayed using styled cards. Each card represents weather data for a specific location.

---

#### 4.2 Responsive Layout

* **Task:** Ensure layout adapts to different screen sizes.

**Solution:**

Flexbox was used to create a responsive design, especially for displaying multiple saved locations.
